System Overview

Claude Agent System (File-Based)

  • 4 specialized Claude agents: PM, Architect, Security, Developer
  • File-based asynchronous coordination
  • Persistent context across sessions
  • Clear audit trail of all decisions
  • Task delegation and escalation protocols

AI Review Board System (MCP-Based)

  • 3 AI nodes: Gemini, Codex (ChatGPT), Claude
  • Real-time dispatch via MCP protocol
  • Read-only access to /workspace (projects)
  • Subscription-based auth (Gemini, Claude)
  • Second opinions, code reviews, research

Use Case Differentiation

  • Claude Agents: Complex multi-phase projects
  • AI Review Board: Quick consultations, validations
  • Combined: Agent escalates to swarm for second opinion
  • Gemini: Primary for external validation + internet search
  • Codex: Alternate when Gemini's answer unsatisfactory

Claude Agent Communication Flow

1

PM Receives Request

User request comes through Claude Code CLI. PM agent breaks down the task and creates task-breakdown.md with assignments for each specialist agent.

2

Architect Designs Solution

Reads PM's task assignment, evaluates technologies, and writes Architecture Decision Records (ADRs) in decisions/{project}/architecture.md

3

Security Configures Infrastructure

Reads architect's design, configures network topology, Traefik routing, Keycloak clients. Writes decisions in decisions/{project}/network.md

4

Developer Implements

Reads architect and security specs, implements the solution, deploys, and documents in notes/{project}/deployment.md

5

PM Synthesizes Report

Reads all agent outputs, compiles progress report, and provides final summary to user.

AI Review Board Consultation Pattern

When to Use Gemini (Primary)

  • Second opinions before major decisions
  • Code reviews for quality and patterns
  • Pair programming / debugging sessions
  • Project plan feedback and validation
  • Internet search for current docs/solutions

When to Use Codex (Alternate)

  • Gemini's response doesn't seem right
  • Debugging takes multiple attempts
  • Complex issues needing multiple perspectives
  • Validating or challenging Gemini's answer
  • Third opinion after asking Gemini twice

When to Use Claude (Swarm)

  • Deep analysis requiring complex reasoning
  • Architecture reviews needing nuance
  • Cross-referencing with main Claude instance
  • Long-form documentation generation
  • When Sonnet perspective complements Opus

Technical Implementation

Docker Architecture

  • Each swarm node runs in isolated container
  • FastAPI HTTP adapter wraps CLI tools
  • Non-root 'swarm' user for security
  • Shared mcp-net Docker network
  • Health checks every 30 seconds

MCP Integration

  • MCP server on localhost:9091
  • dispatch_to_reviewboard tool with 15min timeout
  • swarm_health checks all 3 nodes
  • Stdio transport for Claude Code CLI
  • JSON responses with metrics

File Workspace

  • /workspace mounted read-only to all nodes
  • Maps to $HOME/projects on host
  • Swarm can analyze but not modify
  • All changes made by main Claude instance
  • Auth persisted in named volumes

Agent Workspace

  • $HOME/projects/data/claudeagents/ base
  • Mirrored project paths in workspace
  • tasks/, decisions/, notes/, reports/
  • escalations/ for cross-agent handoffs
  • Persistent across sessions
