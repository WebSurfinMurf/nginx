System Overview

What It Does

  • Enables AI instances on different servers to communicate
  • Allows different users' CLI sessions to share context
  • Human-driven: users manually trigger reads/writes
  • Persistent: messages survive session restarts
  • Simple: just markdown files on a shared mount

Use Cases

  • Server admin hands off task to laptop developer
  • Pass context between administrator and websurfinmurf
  • Share research findings across sessions
  • Leave notes for "future self" on different machine
  • Coordinate multi-machine deployments

Key Difference from AI Swarm

  • AI Swarm: Automated MCP dispatch, same session
  • Shared Chat: Human-driven, cross-session
  • AI Swarm: Different models (Gemini, Codex, Claude)
  • Shared Chat: Same model, different locations
  • Both enable multi-perspective collaboration

How It Works

1. Each AI Writes Its Own File

When working on a task, the AI writes notes to its designated file.

# Administrator on linuxserver writes to: /mnt/shared/aichat/linuxserver.administrator.md # websurfinmurf on laptop writes to: /mnt/shared/aichat/laptop.websurfinmurf.md

2. Other AIs Can Read All Files

User can ask their AI to read another user's file for context.

# User prompt: "Read /mnt/shared/aichat/linuxserver.administrator.md to see what context the admin left for us" # AI loads the file and has that context

3. Handoff Workflow

Administrator hits a permission limit, writes context for developer.

# Admin writes: "I've configured Keycloak client 'stocktrader'. Client secret is in secrets/stocktrader.env. Developer needs to update the app's OAuth config. See /home/administrator/projects/stocktrader/" # Developer reads and continues the work
