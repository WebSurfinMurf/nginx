Specification Phase

Ideas become structured spec.md documents with goals, constraints, and acceptance criteria. YAML front-matter enables automation.

Planning Phase

Architect agent creates plan.md with technical decisions, data flows, and dependencies. Security reviews before approval.

Implementation Phase

Tasks generated as individual files. Developer agents work in parallel using git worktrees. Every commit linked to a task.

Implementation Phases

Weeks 1-2

Phase 1: Spec Bot

Install Spec-Kit CLI, create /specify skill, set up n8n workflow for GitLab Issue to Spec generation. User creates ideas, agent generates structured specs.

Weeks 3-4

Phase 2: Coder Bot

Enhanced Developer Agent with task pickup, git worktree support for parallel development, basic QA for test generation. Tasks auto-assigned, code committed to feature branches.

Weeks 5-8

Phase 3: Closed Loop

Full QA with GitLab CI integration, Security Agent with SAST/DAST, peer review workflows, escalation policies. 80% tasks complete without human intervention.

Weeks 9-12

Phase 4: Dashboard & Analytics

Custom AutomateDev Dashboard (FastAPI + React), traceability graph in ArangoDB, burndown charts, agent activity feed, token usage tracking.

Key Design Principles

  • GitLab as Single Source of Truth - Specs, code, and issues all in one place
  • Phased Rollout - Start with specs, add coding, then full automation
  • Human Approval Gates - AI does the work, humans approve the merges
  • Leverage Existing Infrastructure - 78 containers already deployed
  • [NEEDS CLARIFICATION] Markers - Prevent AI hallucination on ambiguous requirements
