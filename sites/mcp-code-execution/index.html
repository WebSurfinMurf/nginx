üìä Before vs. After Architecture

‚ùå BEFORE: Direct Tool Calling

  • 150,000 tokens loaded upfront
  • All 63 tool definitions always in memory
  • Large datasets (10,000 rows) flow through model
  • Same data written multiple times
  • Cannot save reusable workflows
  • All intermediate data visible to model
  • No complex control flow (loops, retries)

‚úÖ AFTER: Code Execution Mode

  • 2,000 tokens typical usage
  • Load only 3-7 tools per request
  • Filter data in sandbox (only results to model)
  • Process data once in execution environment
  • Save code as reusable skills
  • Tokenize sensitive data automatically
  • Full TypeScript/Python control flow

Code Example: Context Efficiency

// OLD: 10,000 rows through model context TOOL CALL: timescaledb.execute_query(...) ‚Üí returns 10,000 rows (ALL in context) ‚Üí model manually filters ‚Üí high token cost // NEW: Filter in execution environment import * as timescaledb from './servers/timescaledb'; const allData = await timescaledb.execute_query({ query: "SELECT * FROM metrics WHERE timestamp > NOW() - INTERVAL '1 day'" }); // Filter in sandbox (not in model context) const critical = allData.rows.filter(m => m.value > 100); // Model only sees summary console.log(`Found ${critical.length} critical metrics`); console.log(critical.slice(0, 5)); // Show first 5 only

  • Data flow policy engine
